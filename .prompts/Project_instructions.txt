## **DEVELOPMENT PRINCIPLES**

### **Server vs Client**

* Default: **Server Components**
* Use Client Components ONLY for:
  * Interactivity
  * Browser APIs
  * Real-time subscriptions

> Important :рџ‘‰ If user says **"exportable"**, then:
> Use **fully client-side compatible architecture**

---

### **Data Handling**

* Always:
  * Validate with `zod`
  * Infer types from schema

* Use:
  * Typed fetch wrappers
  * Typed server actions

---

### **React Query Rules**
* Feature-scoped queries only
* Separate:
  * `keys`
  * `queries`
  * `mutations`
* Global provider в†’ `shared/lib/providers`

---

### **Security (STRICT)**

* Validate ALL inputs (client в‰  trusted)
* Use:
  * RBAC / ABAC via Appwrite
  * Secure cookies (`HttpOnly`, `Secure`)
  * CSRF protection
  * Rate limiting (Server Actions)

 > вќЊ Never expose secrets to client

---

### **Performance**

* Use:
  * `fetch` caching (`revalidate`, `no-store`, etc. or react-query if applicable)
  * Streaming UI
  * Lazy loading (`next/dynamic`)

* Avoid:
  * N+1 queries
  * Unnecessary client components

---

### **Environment Handling**

* `.env` must be:
  * Validated at runtime
  * Typed
